1. Purpose & Scope

This SOP defines the process for reconciling GL accounts to ensure accuracy and completeness of financial records. The reconciliation process matches GL balances with supporting documentation and identifies discrepancies for investigation and resolution.

Scope: This SOP applies to all GL accounts in the R2R process, including balance sheet accounts (assets, liabilities, equity) and income statement accounts at the account code level.

2. Key Responsibilities

Role Responsibilities
Accounting Analyst Execute reconciliation, identify variances, document adjustments
Senior Accountant Review reconciliation, approve adjustments, investigate significant variances
Finance Manager Own reconciliation completion, escalate unresolved variances
Controller Final review and approval for month-end close

3. Pre-Reconciliation Setup

Step 1: Gather Documents

Collect the following documents for each GL account to be reconciled:

  • GL account extract from accounting system (detailed ledger with all transactions)
  • Supporting documentation (invoices, receipts, bank statements, contracts)
  • Prior month reconciliation (to understand ongoing items)
  • Internal transfer notices and journal entry documentation
  • Any third-party confirmations (bank confirmations, customer confirmations)

Step 2: Identify Account Ownership

  • Determine who is responsible for reconciling the account
  • Verify that GL account is within scope for month-end close
  • Confirm account is active (if inactive, document reason)

Step 3: Set Up Reconciliation Template

  • Open the standard GL reconciliation template (Excel or system template)
  • Input GL account code, account name, and prior month balance
  • Pull GL balance as of month-end cutoff from accounting system
  • Pull supporting documentation balance(s)

4. Reconciliation Execution

Phase A: Identify Reconciling Items

Step 4: Compare GL Balance to Supporting Docs

Identify differences between GL balance and supporting documentation balance:

  1. GL Balance: $847,352
  2. Supporting Doc Balance: $845,125
  3. Variance: $2,227 (need to investigate)

Step 5: Categorize Reconciling Items

Classify variances into one of the following categories:

  • Timing Differences: Items in GL but not yet in supporting docs (or vice versa) due to processing delays
  • Rounding Differences: Minor discrepancies due to currency conversion or calculation rounding
  • Posting Errors: Incorrect GL posting amount, account, or date
  • Documentation Errors: Error in supporting documentation (invoice error, duplicate)
  • Missing Items: Transaction recorded in one place but not the other

Phase B: Investigate Variances

Step 6: Investigate Each Variance

  1. Review GL Transaction Detail: Identify the specific GL transaction(s) causing the variance
  2. Trace to Source Document: Find the original invoice, receipt, or transaction proof
  3. Identify Root Cause: Determine why the variance exists
  4. Document Finding: Record the investigation in the reconciliation workpaper

Example Investigation

Variance Found: $1,500 in GL that doesn't match supporting docs

Investigation:

  • GL shows invoice #INV-2024-5678 for $1,500 dated Jan 30
  • Supporting doc shows invoice received on Feb 2
  • Root Cause: Timing difference - invoice accrued in January GL but payment received in February
  • Resolution: This is a timing difference and reconciles between the months

Phase C: Record Reconciliation

Step 7: Document All Reconciling Items

In the reconciliation template, record:

  1. Description of each reconciling item
  2. Amount of the variance
  3. Category (timing, rounding, posting error, etc.)
  4. Expected resolution date (if not immediate)
  5. Owner responsible for resolution
  6. GL posting date (if adjustment is needed)

5. Resolution of Variances

Timing Differences (No Action Required Now)

Timing differences typically resolve in the following month and do not require immediate GL adjustment. Examples:

  • Invoice recorded in GL but not yet received (accrual)
  • Check recorded in GL but not yet cleared by bank
  • Credit memo in transit

Rounding Differences (Document Only)

Rounding differences of $1-2 typically don't require adjustment if materiality threshold is not met. Document and move on.

Posting Errors (Adjust GL)

If a posting error is identified, create a correcting GL entry:

  1. Prepare correcting journal entry with proper supporting documentation
  2. Obtain approval from Senior Accountant
  3. Post correcting entry to GL with reference to original transaction
  4. Update reconciliation to show resolution

6. Review & Approval Process

Step Owner Approval Criteria
1. Initial Reconciliation Accounting Analyst GL balanced to supporting docs; all variances identified and categorized
2. Review Senior Accountant Variances appropriately investigated; adjustments properly supported
3. Approval Finance Manager Reconciliation complete; no unresolved significant variances
4. Final Sign-off Controller Ready for financial statement inclusion

7. Common Issues & Solutions

Issue Solution
GL doesn't balance to bank statement Check for outstanding checks, deposits in transit, bank fees, interest
Cannot locate supporting documentation Request from initiating department; check archived files
Large unexplained variance Escalate to Finance Manager; may indicate system error
Duplicate transactions in GL Investigate with IT; create reversing entry if confirmed
Reconciliation takes too long Review GL detail for unusual transactions; delegate to multiple analysts

8. Completion & Sign-Off

Final Steps:

  1. Ensure all variances are documented and resolved
  2. Obtain sign-off from Senior Accountant and Finance Manager
  3. File reconciliation workpapers in designated folder (electronic or physical)
  4. Update reconciliation status in tracking sheet
  5. Notify Controller that account is ready for financial statement close
